АНАЛІЗ ПРЕДМЕТНОЇ ОБЛАСТІ
Аналіз вимог та інформаційних потреб
Вимоги користувачів до бази даних (БД) для інформаційної системи обслуговування клієнтів у будівельному супермаркеті можуть виглядати так:
1. Запит на перегляд списку товарів:
•	Інтенсивність: Висока
•	Об'єм даних: Малий
•	Опис: Користувачі хочуть бачити список доступних товарів разом з назвами, цінами та кількістю на складі.
2. Запит на пошук товарів за ключовим словом:
•	Інтенсивність: Середня
•	Об'єм даних: Малий до Середнього
•	Опис: Користувачі хочуть мати можливість шукати товари за ключовими словами або фразами та бачити результати пошуку.
3. Запит на перегляд інформації про конкретний товар:
•	Інтенсивність: Висока
•	Об'єм даних: Малий
•	Опис: Користувачі хочуть бачити деталі товару, включаючи опис, ціну, кількість на складі та інші характеристики.
4. Запит на створення нового замовлення:
•	Інтенсивність: Середня
•	Об'єм даних: Малий до Середнього
•	Опис: Користувачі хочуть додавати товари до свого замовлення та вказувати кількість товарів.
5. Запит на перегляд історії замовлень користувача:
•	Інтенсивність: Середня
•	Об'єм даних: Середній
•	Опис: Користувачі хочуть бачити список своїх замовлень, включаючи статуси та дати.
6. Запит на оновлення статусу замовлення:
•	Інтенсивність: Середня
•	Об'єм даних: Малий
•	Опис: Менеджери хочуть мати можливість оновлювати статуси замовлень (наприклад, "В обробці", "Відправлено", "Доставлено").
7. Запит на додавання нового клієнта до системи:
•	Інтенсивність: Низька
•	Об'єм даних: Малий
•	Опис: Менеджери хочуть мати можливість додавати нових клієнтів до системи разом з їхніми основними контактними даними.

8. Запит на перегляд статистики продажів:
•	Інтенсивність: Середня
•	Об'єм даних: Великий
•	Опис: Адміністратори хочуть мати можливість переглядати статистику продажів за певний період часу, включаючи суми продажів за категоріями товарів.
9. Запит на редагування інформації про товар: 
•	Інтенсивність: Середня 
•	Об'єм даних: Малий до Середнього 
•	Опис: Менеджери хочуть мати можливість змінювати інформацію про товар, включаючи ціну та кількість на складі.

Виявлення інформаційних об’єктів і зв’язків між ними
1.	Клієнт (Client):
•	ID клієнта (Client_ID) - первинний ключ
•	Ім'я (First Name)
•	Прізвище (Last Name)
•	Адреса (Address)
•	Номер телефону (Phone Number)
•	Електронна пошта (Email)
2.	Товар (Product):
•	ID товару (Product_ID) - первинний ключ
•	Назва товару (Product Name)
•	Опис товару (Product Description)
•	Ціна (Price)
•	Кількість на складі (Quantity in Stock)
3.	Замовлення (Invoise):
•	ID замовлення (Invoise_ID) - первинний ключ
•	Дата замовлення (Order Date)
•	Статус замовлення (Order Status)
•	Клієнт (Client_ID) - зовнішній ключ, пов'язаний з ID клієнта
•	Менеджер (Manager_ID) - зовнішній ключ, пов'язаний з ID менеджера
4.	Деталі замовлення (Order Details):
•	ID деталі замовлення (OrderDetail_ID) - первинний ключ
•	Замовлення (Invoise_ID) - зовнішній ключ, пов'язаний з ID замовлення
•	Товар (Product_ID) - зовнішній ключ, пов'язаний з ID товару
•	Кількість (Quantity)
•	Загальна вартість (Total Price)
5.	Склад (Store):
•	ID складу (Store_ID) - первинний ключ
•	Вмісткість (Store_Capacity) 
•	ID продавця (Seller_ID) - зовнішній ключ, пов'язаний з ID продавця
•	Інвентар (Store_Equipment)
6.	Продавець (Seller):
•	ID продавця (Seller_ID) - первинний ключ
•	Замовлення (Invoise_ID) - зовнішній ключ, пов'язаний з ID замовлення
•	Ім’я (Seller_Name)
•	Адреса (Seller_Address)



Зв'язки між таблицями в базі даних можна розписати наступним чином:
1.	Зв'язок між таблицею "Клієнт" і "Замовлення":
•	Клієнт (Client) може мати багато замовлень (One-to-Many зв'язок).
•	В таблиці "Замовлення" є зовнішній ключ "Client_ID", який посилається на ID клієнта в таблиці "Клієнт".
2.	Зв'язок між таблицею "Товар" і "Замовлення" через "Деталі замовлення":
•	Товар (Product) може бути включений у багато різних замовлень через таблицю "Деталі замовлення" (Many-to-Many зв'язок).
•	Таблиця "Деталі замовлення" містить зовнішні ключі "Product_ID" та "Order_ID", які посилаються відповідно на ID товару і ID замовлення в таблицях "Товар" і "Замовлення".
